* 블럭(Block) 요소 : 작성한 순서대로 위에서 아래로 쌓이는 형태

<h1> ~ <h6> : 제목을 정의하는 태그 (Heading Tag)
> 문서의 구조를 정의하고 검색 엔진 최적화(SEO)에 중요한 역할
> <h1> ~ <h6> 순서대로 사용 권장
> <h1> : 페이지에서 한 번만 사용 가능 - 로고
> <h2> : 섹션에 해당하는 제목

<p> : 단락을 정의하는 태그 (일반적 텍스트)

<blockquote> : 긴 인용문을 나타내는 태그
> 앞쪽 여백이 있음 
    <pre> : 스페이스바, 탭키, 엔터 등이 그대로 적용되어 나타남
    > br 태그를 사용하지 않아도 줄바꿈이 가능하다
    > 잘 사용하지 않는 이유 : 일반 글자와 텍스트 스타일이 다름, 유지보수 힘듬 (ex. 스페이스의 간격을 알 수 없음)

* 인라인(Inline) 요소 : 옆으로 나열되어 나타나는 형태

<b> : 텍스트를 굵게 표현시 사용
> 시각적인 스타일 일뿐 의미를 가지고 있지 않음
<strong> : 텍스트를 굵게 표현시 사용
> 의미론적 : 경고, 주의사항과 같이 중요한 내용을 강조
> 웹 접근성을 고려하여 사용

<i> : 텍스트를 이텔릭체 표현시 사용
> 의미론적 중요성은 없음
<em> : 텍스트를 이텔릭체 표현시 사용
> 의미론적 : 주위 텍스트에 비해서 강조할 때

<q> : 짧은 인용문을 나타내는
> 앞 뒤에 쌍따옴표 표현
> 의미론적 : 저자의 생각

<mark> : 텍스트에 형관펜 효과를 주어 강조할 때 사용
> 의미론적 : 강조

<span> : css등 이용하여 글자를 꾸밀 때 사용
> 시각적으로 나눠서 스타일을 지정하고 싶을 때 사용 > 의미론적인 중요성은 없음

<u> : 글자에 밑줄 표현시 사용
> 의미론적 중요성은 없음
<ins> : 글자에 밑줄 표현시 사용
> 의미론적 : 강조할 때

<s> : 글자에 관통하는 선 표현시
> 의미론적 중요성은 없음
<del> : 글자에 관통하는 선 표현시
> 의미론적 : 문서에서 삭제된 텍스트나 지워야할 부분을 강조할 때

<sup> : 글자의 위로 붙는 위첨자

<sub> : 글자의 아래로 붙는 아래첨자

목록태그 (Block 요소 성질)

<li> : ol, ul사이 목록태그 (list item)

- 목록 태그의 속성 > 중첩이 가능하며, 개수의 제한이 없다

- type=""으로 속성 입력
> 기호변경 disc, circle, square, none
> 순서변경 a, A, i, I

- start="시작순서"

<ul> : 순서가 없는 목록 태그
> 각 항목 앞에 불릿기호
> 메뉴 제작시 사용

  • menu-1
  • menu-2
  • menu-3
  • menu-4
  • menu-5
  • Alt + Shift + 방향키 = 복사

<ol> : 순서가 있는 목록 태그
> 순서가 있는 항목

  1. 순서
  2. 순서
  3. 순서
  4. 순서
  5. 순서

<dl> : 정의 목록 태그
> 정의 목록을 설명한 용어와 뜻을 표현 > 전자명함, <dt> : 설명할 용어나 이름 정의 <dd> : 해당 용어에 대한 설명이나 정의를 제공

<a> 태그

> 웹 페이지의 연결, URL 연결 시 사용 >> 다른 페이지의 전환(메뉴표현)
> Inline 요소

초록창

- target="" : 링크를 클릭 시 열리는 위치를지정
> _self : 현재 창에서 전환 / 기본값
> _blank : 새 창에서 전환
> _parent : 부모 프레임에서 전환 (여러 개의 프레임이 겹쳐져 있을 때 사용)
> _top : 최상위 프레임에서 전환 (여러 개의 프레임이 겹쳐져 있을 때 사용)
