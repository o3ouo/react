img 태그

> 이미지를 나타내줄 수 있는 태그
> 단독 태그
> Inline 성질


* 이미지 파일 확장자 (대표적으로 많이 사용)

- jpg
> 가장 대표적인 이미지 표현 방식
> 이미지의 색상을 가장 잘 표현할 수 있는 확장자
> 투명한 배경 x
> 용량이 가벼움

- png
> 이미지의 색상을 유지
> 투명한 배경을 나타낼 수 있는 확장자
> 이미지 표현이 jpg보다 좋지 않음
> 많이 깨짐
> 상품 넣을 때 많이 사용
> jpg보다 용량이 무거움

- gif
> 움직이는 이미지 표현시 사용
> 투명한 배경 가능
> RGB 색상 표현 방식
> 꾸며주는 효과를 넣는 이미지를 넣을 때 사용

- webp
> 기존의 이미지를 웹 환경에 최적화 시켜 압축 시킨 확장자
> 구글에서 개발
> 용량을 줄일 수 있다
> 이미지를 띄울 때 업로드 속도를 빠르게 하며 (페이지 로딩 시간 줄일 수 있음)
> 검색 엔진(SEO)에 유리
> 호환성 문제로 모든 브라우저에서 사용되지는 않는다
> 편집 소프트웨어 지원 제한 (압축 된 파일이기 때문에 포토샵 등으로 크기 조절 불가능)
> 기존 확장자로 다시 바꿔서 편집해야 함
> 대표적으로 나무위키에서 사용


* img 태그 속성 src와 a 태그 속성 href의 차이점

- src
> source의 약자
> 외부 리소스를 html 문서에 직접 포함할 때 사용 (저장을하고 거쳐나가는 방식)
> 즉시 로드하여 문서에 포함시키기 때문에 페이지 파싱(페이지 출력)이 일시적으로 중단될 수 있다
> image, script, 영상, iframe
> img 태그를 쓸 때에는 src에 url을 연결할 수 있다
>> 하지만 잘 사용하지 않음
>> 해당 주소에서 가져온 이미지를 사용할 시 기존 주소에서 이미지를 내리면 내가 사용한 곳에서도 내려간다
>> 저작권 문제

- href
> 현재 문서와 외부 리소스 간의 링크를 설정하는 데 사용
> 가지고 와서 연결만 시키는 것
> html 문서에 직접적으로 저장시키는 것이 아니라 외부와의 연결만을 하는 역할
> 즉시 로드하여 문서에 포함시키기 때문에 페이지 파싱(페이지 출력)이 일시적으로 중단될 수 있다
> link, stylesheet 연결 시 사용
> 페이지 파싱(페이지 출력)이 중단되지 않고 필요할 때만 리소스를 가져와서 사용


* img 태그 속성에는 필수적으로 alt 속성을 입력해야 한다.

- alt
> 이미지 설명
> 대체문
> 접근성에 의해 필수적으로 사용해야 한다


* 이미지를 감싸는 태그 figure

> 이미지, 영상, 그래픽 등의 멀티미디어 콘텐츠와 그에 대한 설명을 그룹화 하는 데 사용
> 검색엔진 최적화에 도움을 많이 줌
> 블럭요소의 성질을 가지고 있음
> 각각의 이미지를 한 장씩 넣어서 사용하는 것이 좋다
> 대부분의 브라우저에서 지원하므로 웹 페이지의 접근성과 검색엔진최적화(SEO) 향상에 기여
> 이미지 등을 담는 역할을 하는 시멘틱 태그

* figcaption

> 콘텐츠에 대한 설명
> 선택적으로 사용
> 콘텐츠의 첫 번째 또는 마지막에 배치해야 한다
