* form
> 웹페이지에서의 입력 양식의 그룹
> 원하는 정보를 연결하여 전달받기 위한 양식
* action
> 폼 데이터를 제출할 때 데이터를 보낼 url 지정
> 생략할 수 있음
* method
> 폼 데이터를 전송할 때 사용할 HTTP 메서드를 정의
* get : 데이터를 그대로 보냄
>> 주소 표시줄에 사용자가 입력한 내용이 그대로 표시
>> 보안상 좋지 않음
>> url 길이가 제한되어 있음
>> 반복 요청을 많이 하는 사이트에서 사용
>> 전자상거래, 날씨, 제품검색
* post
>> 내부적으로 보이지 않게 전송
>> 파일 업로드에 사용
>> 담아서 사용하기 때문에 url 노출이 없고 get 보다 보안성이 높음
>> 큰 양의 데이터를 처리할 수 있다
>> 정보를 비공개해서 보내야할 때
* name
> form의 이름
> form을 식별하기 위한 용도

입력필드

: 사용자의 정보나 회원가입, 검색 등과 같은 다양한 정보를 입력 받을 때 사용하는 형태
